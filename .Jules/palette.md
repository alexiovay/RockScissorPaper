## 2024-04-26 - Keyboard Access for Icon-Only Non-Button Elements
**Learning:** In React applications, developers sometimes use non-interactive elements like `<li>` or `<div>` with emojis/icons to act as buttons (e.g., weapon selections in a Rock Paper Scissors game). These lack semantic meaning, keyboard support (tabbing, enter/space to activate), and context for screen readers. Simply adding an `onClick` handler is insufficient for accessibility.
**Action:** When encountering icon-only interactive elements using non-semantic tags, always convert them to behave like buttons by adding `role="button"`, `tabIndex={0}`, an `onKeyDown` handler to simulate click via Enter/Space, and an `aria-label` describing the action. Additionally, update the CSS to include a clear `:focus-visible` state and ensure `display: inline-block` (or similar) is used so the focus ring properly wraps the element.
