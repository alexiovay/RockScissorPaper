## 2024-04-19 - Ensure semantic structure and live updates for custom interactive components
**Learning:** Custom interactive elements (like icon-only `<li>` items acting as buttons) require proper roles (`role="button"`), tab indexing (`tabIndex={0}`), and keyboard event handling (`onKeyDown` for Enter/Space) for accessibility. Furthermore, dynamic game outcomes injected directly into the DOM aren't naturally announced to screen readers unless the container uses `aria-live="polite"`. Also inputs relying entirely on placeholders instead of labels need `aria-label` attributes.
**Action:** Next time, ensure all clickable, non-button elements are keyboard accessible and correctly labeled. Always apply `aria-live` to regions that present dynamic notifications or state changes.
